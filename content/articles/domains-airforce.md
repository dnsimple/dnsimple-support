This article summarizes requirements for **.airforce** domains operated by the Donuts registry, based on [CentralNic Reseller TLD documentation](https://kb.centralnicreseller.com/domains/tlds/airforce).

## Registry restrictions {#restrictions}

### Sanctions (OFAC) validation

The registry validates contact addresses against U.S. OFAC sanctions rules. Certain **country and state/province** combinations are not allowed. If your contact uses a restricted combination, registration or updates may be rejected.

### Premium and reserved names (allocation token)

Some names are held as reserved or premium by the registry. When Donuts assigns an **allocation token** for a specific name, you must provide that token when registering so the registry can release the reservation. If you were given a token by Donuts or a partner program, enter it when DNSimple prompts for registry extended attributes (allocation token).

## Transfers and auth codes {#transfers}

Transfer uses an authorization code from the losing registrar. Auth codes for this TLD must:

- Be **8–32** characters long, and  
- Include at least one **letter**, one **number**, and one **special** character from the set allowed by the registry (see registrar documentation for the exact character class rules).

New registrations are **transfer-locked at the registry for 60 days** after creation.

## Internationalized domain names (IDN) {#idn}

.AIRFORCE supports multiple scripts. The registry expects the appropriate **IDN language tag** when registering internationalized labels. See [IANA IDN tables](https://www.iana.org/domains/idn-tables) for permitted code points.
